## Cursor Rules

### Development Rules
Define the delivery stack and guardrails for implementation.

- **Languages to use?**
  Primary languages used are JavaScript, HTML/CSS, and potentially Python for backend services.

- **Frontend framework preference?**
  The frontend framework preference is React, given its popularity and versatility.

- **Styling approach?**
  Styling will be handled using CSS with a preprocessor like Sass for efficiency and consistency.

- **State management plan?**
  State management will be handled using Redux for complex state management needs.

- **Backend or data layer?**
  The backend will be built using Node.js, potentially with Express for routing and API management.

- **APIs to connect and which to mock?**
  API connections include the GroqAPI for core functionality and potentially other third-party services for additional features.

- **Testing expectations?**
  Testing will involve unit tests, integration tests, and UI tests to ensure comprehensive coverage.

- **Security guidelines?**
  Security guidelines include encryption for sensitive data, secure authentication practices, and regular security audits.

- **Performance considerations?**
  Performance considerations include optimizing database queries, minimizing HTTP requests, and using caching where appropriate.

- **File organization conventions?**
  File organization will follow a modular approach, with separate folders for components, services, and utilities.

- **Commit conventions?**
  Commit conventions will follow the standard GitHub guidelines, with descriptive commit messages and appropriate use of issues and pull requests.
